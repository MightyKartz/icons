# 贡献指南

感谢您对Icons项目的关注！我们欢迎所有形式的贡献。

## 🤝 如何贡献

### 报告问题
- 使用 [GitHub Issues](https://github.com/MightyKartz/icons/issues) 报告bug
- 提供详细的重现步骤
- 包含系统环境和错误日志

### 提出功能建议
- 在Issues中使用"enhancement"标签
- 详细描述功能需求和使用场景
- 说明为什么这个功能有价值

### 提交代码
1. Fork项目到您的GitHub账号
2. 创建功能分支：`git checkout -b feature/amazing-feature`
3. 提交更改：`git commit -m '添加某个功能'`
4. 推送分支：`git push origin feature/amazing-feature`
5. 创建Pull Request

## 📋 开发规范

### 代码风格

#### TypeScript/React
- 使用TypeScript严格模式
- 遵循ESLint规则
- 组件使用PascalCase命名
- 文件使用kebab-case命名

#### Python
- 遵循PEP 8规范
- 使用类型提示
- 函数使用snake_case命名
- 类使用PascalCase命名

#### Swift
- 遵循Swift官方代码规范
- 使用SwiftLint检查
- 类型和协议使用PascalCase
- 变量和函数使用camelCase

### 提交信息规范
```
类型(范围): 简短描述

详细描述（可选）

相关问题: #123
```

类型包括：
- `feat`: 新功能
- `fix`: 修复bug
- `docs`: 文档更新
- `style`: 代码格式调整
- `refactor`: 代码重构
- `test`: 测试相关
- `chore`: 构建工具或辅助工具的变动

### 测试要求
- 新功能必须包含测试
- 测试覆盖率不应降低
- 确保所有测试通过

## 📝 Pull Request流程

### 提交PR前检查
- [ ] 代码符合项目规范
- [ ] 所有测试通过
- [ ] 添加了必要的测试
- [ ] 更新了相关文档
- [ ] 提交信息清晰明确

### PR模板
```markdown
## 变更类型
- [ ] Bug修复
- [ ] 新功能
- [ ] 代码重构
- [ ] 文档更新
- [ ] 其他：_______

## 变更描述
简要描述此PR的变更内容。

## 测试
描述如何测试这些变更。

## 相关Issue
关闭的Issue编号：#123

## 检查清单
- [ ] 代码通过所有测试
- [ ] 代码符合项目规范
- [ ] 自我审查了代码
- [ ] 添加了必要的注释
- [ ] 更新了相关文档
```
